# Code of Conduct

## Our standard

We are a correctness-first engineering community. Every interaction —
in issues, PRs, debates, code reviews — is expected to be:

- **Technical.** Arguments carry weight based on evidence (grep, read,
  tests, benchmarks), not status.
- **Specific.** "This is wrong" is not feedback. "This is wrong because
  line X violates invariant Y" is feedback.
- **Respectful.** Discuss ideas, not people. Assume good faith.
- **Concise.** A clear sentence beats a clear paragraph.

## Unacceptable behavior

- Personal attacks, harassment, or discrimination of any kind
- Sexualized language or imagery
- Publishing private information without permission
- Sustained disruption of debates, discussions, or review
- Other conduct that would be inappropriate in a professional setting

## Reporting

Maintainers listed in `.github/CODEOWNERS` handle conduct concerns.
Reports via GitHub issue (if public) or direct contact to a maintainer
(if sensitive). Reports are acknowledged within 7 days.

## Enforcement

Maintainers may:

- Remove, edit, or reject contributions that violate this Code
- Temporarily ban contributors for repeated violations
- Permanently ban contributors for severe or repeated violations

Enforcement decisions are documented with rationale.

## Scope

This Code applies to:
- The repository (issues, PRs, discussions, code, commits)
- Direct communications with maintainers in connection with the project
- Public events where someone represents this project
