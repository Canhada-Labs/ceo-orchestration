# ADR-150 — Commit-signing policy: signed-tag ratification + tiered signing requirement

- **Status:** PROPOSED (S227, 2026-06-10; Owner pre-ratified the direction in-session — formal ACCEPT at the next promotion sweep)
- **Context:** REPORT-S225 finding **E7-F1 [P1]** — "GPG signing discipline unaudited; 44 of 116 delta commits UNSIGNED on main". S227 forensics (workflow `wf_6caca3aa`) reproduced the count exactly at the S225 merge point and established the ground truth: 46 unsigned in the current delta window (19 docs/plans-only, 14 non-canonical code, 13 CLAUDE.md closeout edits), **zero** unsigned commits touching `_lib/**`, `settings.json`, `PROTOCOL.md`, `team.md`, or core `SKILL.md`; all-time 1133/1513 (75%) unsigned. No repo document required per-commit signing — `docs/BRANCH-PROTECTION.md` marks it "(optional)", and ADR-003 Path C's compensating control is the **Owner-GPG sentinel on canonical EDITS** (authoring-time), not commit signatures. The actual defect was an UNWRITTEN policy.
- **Decision (Owner, S227):**
  1. **Signed-tag ratification.** The Owner creates ONE GPG-signed tag (`ratify-provenance-s227`) on current main. A tag signature covers the commit hash, which Merkle-covers every ancestor — one signature retroactively attests integrity + endorsement of the full history. Repeatable at future milestones (`ratify-provenance-sNNN`).
  2. **Tiered signing requirement (prospective):**
     - **MUST be Owner-GPG-signed:** ceremony commits applying canonical-guarded paths (`_lib/**`, `.claude/hooks/*.py`, `settings.json`, `CLAUDE.md`, `PROTOCOL.md`, `team.md`, core `SKILL.md`), release/tag commits, and ADR promotions.
     - **MAY be unsigned:** docs/plans/closeout commits, non-canonical code + tests, PR work commits — provenance for these rests on the sentinel chain + HMAC audit chain + the periodic ratification tag.
  3. Per-commit signing enforcement (hook/CI) is explicitly **rejected** — Claude-authored closeout commits cannot be signed without Owner interaction per commit; the friction buys nothing the sentinel chain doesn't already provide.
- **Consequences:** E7-F1 closes as documented-by-design + cryptographically ratified. `docs/BRANCH-PROTECTION.md` §Commit-signing policy is the operational reference. Future audits MUST evaluate signing against this tiered policy, not against a per-commit ideal.
- **Residual:** unsigned history remains individually unattributed (a compromised credential could have authored any pre-tag commit); accepted — the ratification tag records the Owner's post-review endorsement, and the sentinel/HMAC chains carry the per-edit authority.
