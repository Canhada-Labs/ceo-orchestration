# PLAN-155 Wave 1 — host-adapter implementation notes (SENT-CX-A ceremony input)

Companion to the staged overlay `PLAN-155/staged/wave-1/` (adapter +
seam + golden/drift/characterization tests + fixtures). Written by the
Wave 1 host-adapter task, 2026-07-10. Normative wire facts come from the
recorder artifacts (`normalization-notes.md`, `failure-semantics-matrix.md`,
`trust-keying-A6.md`); this file records the IMPLEMENTATION decisions the
ceremony and ADR-161 need.

## Decisions taken (and why)

1. **A2 coherence-gate home = seam + adapter side; `contract.py`
   UNTOUCHED.** The env-level half (explicitly-set-but-unresolvable
   `CEO_HOOK_ADAPTER`) lives in `_lib/adapters/__init__.py:resolve()`
   (returns a `_FailClosedAdapter` whose egress always denies in BOTH
   harness vocabularies); the envelope-level half (cross-harness /
   unparseable-apply_patch detection) lives in `codex.py` host-mode
   `read_event` (`raw_payload['ceo_coherence_error']`) with the deny
   override applied at `write_decision(..., event=)` egress. Adding a
   NormalizedEvent field instead would have forced a SPEC/v1
   `normalized_envelope.schema.md` row — SENT-CX-A is signed
   `Amends: none`, so the contract dataclass was deliberately left
   alone (drift-detector test
   `test_codex_host_extras_ride_raw_payload_not_new_fields` pins this).
   Owner action at signing: strike the conditional `contract.py` line
   from SENT-CX-A scope (option "adapter-side" chosen), or keep it and
   land the file byte-identical.

2. **Host output shape is EXPLICIT-ONLY, no module latch.** Host shape
   is selected by `Decision.extra['hookEventName']` or the `event=`
   kwarg (host-wire events carry `raw_payload['ceo_host_wire']`). A
   module-global "last read was host-wire" latch was rejected: hooks
   are one-shot subprocesses so it would work in production, but it is
   order-dependent inside a shared pytest process — exactly the
   sequential-pollution flake class the finish-gate lesson documents.
   CONSEQUENCE the hook-migration task must honor: a migrated hook MUST
   pass the parsed event (or stamp extra) at emit time; a deny emitted
   in the legacy shape is foreign JSON to codex = silent allow (S254
   class). The subprocess positive controls are the proof this never
   happens on the shipped command line.

3. **Legacy `write_decision` output preserved byte-identical** when no
   host context is present — the 37 pre-existing `test_codex_adapter.py`
   tests pass unchanged, and reviewer-context tooling is unaffected.
   The "goldens updated deliberately" item therefore shows up as NEW
   codex `out/` goldens (host shapes), not as edits to old ones.

4. **Normalization choices** (locked by
   `fixtures/adapters/codex/normalized/*.json`, generated by executing
   the staged adapter over the 17 recorded in-fixtures, then reviewed):
   - `phase` carries the wire `hook_event_name` VERBATIM, including
     lifecycle values (`SessionStart`, `Stop`, `UserPromptSubmit`,
     `SubagentStart`, `SubagentStop`) — the wire wins over the `phase`
     argument. Claude-adapter behavior is untouched.
   - `apply_patch` → `tool_name` `Write` (pure Add-File patch) or
     `Edit` (any Update/Delete/Move); `tool_input` is ENRICHED with
     `file_path` (first op), `apply_patch_paths` (FULL list incl. move
     targets — guards must deny if ANY path is guarded),
     `apply_patch_ops`; raw patch text stays in `command`.
   - `spawn_agent` → `Task` with `tool_input.prompt`/`subagent_type`
     aliases (so `check_agent_spawn` sees the Claude shape; PreToolUse
     deny on spawn IS enforcing on 0.139 — e6 transcript).
   - string `tool_response` → `{"output": <str>}` (contract field is
     Dict-typed).
   - codex-only scalars (`turn_id`, `stop_hook_active`,
     `last_assistant_message`, `agent_id`, `agent_transcript_path`,
     `permission_mode`, `source`, `cwd`, `model`) ride `raw_payload`;
     `project` = wire `cwd` (falls back to `CLAUDE_PROJECT_DIR`).
   - `_meta` (fixture provenance) is NOT wire data and is ignored by
     the adapter.

5. **Cross-harness detector is deliberately narrow**: Claude-native
   tool names (`Edit`/`Write`/`MultiEdit`/`NotebookEdit`/`Task`) WITH
   Claude-native `tool_input` keys on PreToolUse/PostToolUse. A plain
   Bash envelope is valid under both harnesses by upstream design and
   must not trip it. Residual (named): a cross-harness envelope that
   avoids those markers is not detected here — the registration/env
   wiring (Wave 2) and positive controls own that surface. Re-verify
   the "codex never emits these tool names" premise at every pin bump
   (substrate-watch re-test list).

6. **Coherence denies speak BOTH vocabularies** (top-level
   `decision: block` + `hookSpecificOutput` deny): under a
   mis-configuration we cannot know which harness is reading. Verified
   harmless on 0.139 (unknown sibling keys beside a valid
   `hookSpecificOutput` are ignored).
